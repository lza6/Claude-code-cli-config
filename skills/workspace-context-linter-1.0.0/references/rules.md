# 规则

## P0 lint 类别

### 重复
当多个核心上下文文件表达相同的稳定规则、偏好或边界且仅进行微小的措辞变化时进行标记。

### 超重
当文件或部分对于始终加载的上下文来说显得太程序化、太长或太详细时进行标记。

### 放错地方了
当内容出现在错误的核心文件角色中时进行标记，例如“TOOLS.md”外部的环境特定详细信息或“MEMORY.md”内部的临时事实。

## Severity guidance
- 高：可能会造成执行模糊性或长期漂移
- medium: increases maintenance cost or context weight noticeably
- 低：清理机会，短期影响有限

## 第一个版本的限制
- 不要自动编辑文件。
- 不要尝试深度语义聚类。
- 首先关注核心文件；将内存扩展视为可选。
