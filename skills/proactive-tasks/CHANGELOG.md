# 变更日志 - 主动任务

该项目的所有显着更改都将记录在该文件中。

## [1.2.0] - 2026-02-12

### 添加 - 第 2 阶段：生产就绪架构

#### WAL 协议（预写日志记录）
- 新的“log_to_wal()”函数在数据修改之前记录关键更改
- 防止上下文中断时数据丢失
- 自动创建 `memory/WAL-YYYY-MM-DD.log` 文件
- 集成到标记进度、记录时间、标记阻止、运行状况检查命令中
- 记录事件类型：PROGRESS_CHANGE、TIME_LOG、STATUS_CHANGE、HEALTH_CHECK

#### SESSION-STATE.md - 活动工作内存
- 每次任务操作时自动更新
- 捕获：当前任务、进度、记录时间、阻碍因素、下一步行动
- 充当当前会话的活动工作内存
- 更新内容：进度变更、时间日志、状态变更
- 包括速度计算（比估计快/慢）

#### 工作缓冲区 - 危险区域捕获
- `working-buffer.md` 模板捕获上下文切断风险期间的更新
- 自动附加所有任务更新（PROGRESS、TIME_LOG、STATUS_CHANGE）
- 压缩时刷新到每日内存文件（`memory/YYYY-MM-DD.md`）
- `flush-buffer` 命令触发手动刷新
- 当上下文在会话中被截断时，防止信息丢失

#### 自愈健康检查
- 新的“健康检查”命令检测并修复损坏的任务状态
- 检测 5 类问题：
  1. 孤立的重复任务（无父目标）
  2.不可能的状态（已完成但进度<100）
  3. 缺少完成时间戳
  4. 损坏次数（实际>>估计）
  5. 未来已完成的任务
- 自动修复 4 个安全类别，标记时间异常以供人工审核
- 运行：`python3 script/task_manager.py health-check`

####自主运行架构
- 两种模式系统：交互式（systemEvent）与自主（独立的agentTurn）
- 文档：带有完整指导的“HEARTBEAT-CONFIG.md”
- 关键原则：不要使用systemEvent进行后台工作
- 包括心跳轮询模式（推荐用于大多数工作）
- 包括独立的 cron 模式（速度报告、清理）
- 记录避免的反模式

#### 进化护栏
- “EVOLUTION.md”，带有新功能的 VFM/ADL 评分框架
- VFM 协议：值频率倍增器（3x HF、3x FR、2x UB、2x SC）
- ADL协议：架构设计阶梯（稳定性>可解释性>可重用性>可扩展性>新颖性）
- 通过阈值：>= 60 VFM 点且 ADL 通过
- 防止特征蠕变，保持生产稳定性

#### 新命令
- `mark-progress <task_id> <percent>` - 使用 WAL 日志记录设置任务进度
- `log-time <task_id> <分钟>` - 记录速度计算所花费的时间
- `mark-blocked <task_id> <reason>` - 标记任务因原因而被阻止
- “健康检查” - 检测并自动修复损坏的状态
- `flush-buffer` - 手动将工作缓冲区刷新到日常内存
