# 清单 (Checklist) 📋

**AI 代理的任务清单管理器。创建、管理和跟踪复杂任务的清单。**

---

## 🎯 何时使用

在以下情况下使用此技能：
- 用户要求创建任务清单
- 用户需要跟踪流程中的多个步骤
- 复杂的多步骤操作需要验证
- 需要质量保证或审查流程
- 入职或设置程序
- 部署前的“飞行前检查”

## ⚡ 快速命令

```bash
# 从模板创建清单
checklist create <template-name>

# 列出可用模板
checklist templates

# 显示当前清单
checklist show

# 将项目标记为已完成
checklist done <item-number>

# 将项目标记为已跳过
checklist skip <item-number> --reason "..."

# 添加自定义项目
checklist add "<item-description>"

# 移除项目
checklist remove <item-number>

# 重置清单
checklist reset

# 导出清单
checklist export
```

## 🔧 实施

### 清单存储

清单存储在 `~/.checklist/` 中：
- `~/.checklist/templates/` - 内置模板
- `~/.checklist/active/` - 当前活跃清单
- `~/.checklist/history/` - 已完成的历史清单

### 模板格式 (JSON)

```json
{
  "name": "部署清单",
  "description": "部署前验证步骤",
  "items": [
    {
      "id": 1,
      "text": "本地运行所有测试",
      "required": true
    },
    {
      "id": 2,
      "text": "审查代码更改",
      "required": true
    }
  ]
}
```

## 🎯 最佳实践

### 1. 从模板开始
使用现有的内置模板 —— 它们针对常见场景进行了优化。

### 2. 保持具体
每个清单项目应当是：
- **可操作的** —— 明确要采取的行动
- **可验证的** —— 可以确认完成
- **原子的** —— 单一任务，而非复合任务

### 3. 跟踪进度
定期更新清单状态：
- 查看剩余项
- 识别阻塞项
- 记录跳过的项目

### 4. 完成后回顾
完成清单后：
- 查看跳过的项目及其原因
- 为下一次改进提供建议
- 导出文档

## ⚠️ 重要提示

### 何时使用清单

✅ **建议使用清单：**
- 多步骤程序
- 复杂的部署
- 安全审查
- 入职
- 质量保证 (QA)

❌ **不要过度使用：**
- 简单的 1-2 步任务
- 无固定步骤的创意工作
- 临时性的小问题

### 与其他技能的整合

清单适用于：
- **任务控制** —— 将清单项目转换为正式任务
- **GitHub** —— 提交前/部署清单
- **健康检查** —— 安全检查表

---

## 🦞 总结

**一句话：当用户需要跟踪多步骤任务时 → 创建清单 → 跟踪进度 → 确保没有遗漏。**
