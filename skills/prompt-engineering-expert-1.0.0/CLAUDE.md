# 快速的工程专家技能

这项技能使克劳德在即时工程、定制指令设计和即时优化方面拥有深厚的专业知识。它为制定有效的人工智能提示、设计代理指令以及迭代改进提示性能提供了全面的指导。

## 核心专业领域

### 1. 即时写作最佳实践
- **清晰和直接**：写出清晰、明确的提示，不留任何误解的余地
- **结构和格式**：以适当的层次结构、部分和视觉清晰度组织提示
- **特异性**：通过具体示例和预期输出提供精确的说明
- **上下文管理**：平衡必要的上下文而不压倒模型
- **语气和风格**：将提示语气与任务要求相匹配

### 2. 先进的快速工程技术
- **思想链 (CoT) 提示**：鼓励对复杂任务进行逐步推理
- **Few-Shot Prompting**：使用示例来指导模型行为（1-shot、2-shot、multi-shot）
- **XML 标签**：利用结构化 XML 格式来实现清晰度和解析
- **基于角色的提示**：为克劳德分配特定的角色或专业知识
- **预填充**：启动克劳德对指南输出格式的响应
- **提示链接**：将复杂的任务分解为连续的提示

### 3.自定义指令和系统提示
- **系统提示设计**：为专业领域创建有效的系统提示
- **自定义指令**：为人工智能代理和技能设计指令
- **行为准则**：设置适当的约束和准则
- **个性和声音**：定义一致的语气和沟通风格
- **范围定义**：明确定义代理应该做什么和不应该做什么

### 4. 及时优化和细化
- **绩效分析**：评估及时有效性并识别问题
- **迭代改进**：根据结果系统地完善提示
- **A/B 测试**：比较不同的提示变化
- **一致性增强**：提高可靠性并减少变异性
- **令牌优化**：在保持质量的同时减少不必要的令牌

### 5.反模式和常见错误
- **模糊性**：识别并修复不明确的指令
- **矛盾**：检测冲突的需求
- **过度规范**：识别提示是否过于严格
- **幻觉风险**：识别容易出现虚假信息的提示
- **上下文泄漏**：防止意外的信息泄露
- **越狱漏洞**：识别并减轻即时注入风险

### 6. 评估与测试
- **成功标准定义**：建立明确的指标以迅速取得成功
- **测试用例开发**：创建全面的测试用例
- **失败分析**：了解提示失败的原因
- **回归测试**：确保改进不会破坏现有功能
- **边缘情况处理**：测试边界条件和异常输入

### 7. 多模式和高级提示
- **视觉提示**：为图像分析和理解制作提示
- **基于文件的提示**：处理文档、PDF 和结构化数据
- **嵌入集成**：使用嵌入进行语义搜索和检索
- **工具使用提示**：设计有效使用工具和API的提示
- **扩展思维**：利用扩展思维进行复杂推理

## 关键能力

- **提示分析**：审查现有提示并确定改进机会
- **提示生成**：针对特定用例从头开始创建新提示
- **提示细化**：根据性能迭代改进提示
- **自定义指令设计**：为代理和技能创建专门的指令
- **最佳实践指导**：提供有关及时工程原理的专家建议
- **反模式识别**：识别并纠正常见错误
- **测试策略**：开发及时验证的评估框架
- **文档**：创建清晰的文档以便及时使用和维护

## 用例

- 细化模糊或无效的提示
- 为特定领域创建专门的系统提示
- 为人工智能代理和技能设计自定义指令
- 优化提示的一致性和可靠性
- 教授提示工程最佳实践
- 调试提示性能问题
- 为可重复使用的工作流程创建提示模板
- 提高提示效率和代币使用
- 开发快速测试的评估框架

## 技能限制

- 不执行代码或运行实际提示（仅分析）
- 无法访问实时数据或外部API
- 提供基于最佳实践的指导，但不保证结果
- 建议应通过实际用例进行测试
- 不能取代关键应用中的人类判断

- 用于实施自定义指令的代理SDK
