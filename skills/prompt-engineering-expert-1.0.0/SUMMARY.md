## 核心专业领域

1. **提示写作最佳实践**
   - 清晰、直接
   - 结构和格式
   - 特异性
   - 上下文管理
   - 语气和风格

2. **先进的快速工程技术**
   - 思维链（CoT）提示
   - 少发提示
   - XML标签
   - 基于角色的提示
   - 预填充
   - 提示链接

3. **自定义指令和系统提示**
   - 系统提示设计
   - 定制说明
   - 行为准则
   - 个性和声音
   - 范围定义

4. **及时优化和细化**
   - 性能分析
   - 迭代改进
   - A/B 测试
   - 一致性增强
   - 代币优化

5. **反模式和常见错误**
   - 模糊性
   - 矛盾之处
   - 超规格
   - 幻觉风险
   - 上下文泄漏
   - 越狱漏洞

6. **评估与测试**
   - 成功标准定义
   - 测试用例开发
   - 故障分析
   - 回归测试
   - 边缘情况处理

7. **多模式和高级提示**
   - 视觉提示
   - 基于文件的提示
   - 嵌入集成
   - 工具使用提示
   - 延伸思考

## 关键能力

1. **提示分析** - 审查并改进现有提示
2. **提示生成** - 从头开始创建新提示
3. **提示细化** - 迭代改进提示
4. **自定义指令设计** - 创建专门的指令
5. **最佳实践指南** - 教授即时工程原理
6. **反模式识别** - 识别并纠正错误
7. **测试策略** - 制定评估框架
8. **文档** - 创建清晰的使用文档

## 如何使用此技能

### 用于即时分析
```
"Review this prompt and suggest improvements:
[YOUR PROMPT]"
```

### 用于提示生成
```
"Create a prompt that:
- [Requirement 1]
- [Requirement 2]
- [Requirement 3]"
```

### 自定义说明
```
"Design custom instructions for an agent that:
- [Role/expertise]
- [Key responsibilities]"
```

### 用于故障排除
```
"This prompt isn't working:
[PROMPT]

Issues: [DESCRIBE ISSUES]

How can I fix it?"
```

## 包含最佳实践

### 要做的 ✅
- 清晰具体
- 提供例子
- 指定格式
- 定义约束

- 忽略边缘情况
- 过度指定约束
- 使用行话而不解释
- 硬编码值
- 忽略错误处理
