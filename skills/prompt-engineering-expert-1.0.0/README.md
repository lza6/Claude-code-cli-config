# 自述文件 - 提示工程专家技能

## 概述

**提示词 (Prompt) 工程专家**技能使克劳德在提示词 (Prompt) 工程、自定义指令设计和提示词 (Prompt) 优化方面拥有深厚的专业知识。这项综合技能为制定有效的 AI 提示词 (Prompt)、设计代理指令以及迭代改进提示词 (Prompt) 性能提供了指导。

## 该技能提供什么

### 核心专业知识
- **提示词 (Prompt) 写作最佳实践**：清晰、直接的提示词 (Prompt)，结构正确
- **高级技术**：思维链、少量示例提示词 (Few-shot)、XML 标签、基于角色的提示词 (Prompt)
- **自定义指令**：系统提示词 (Prompt) 和代理指令设计
- **优化**：分析和完善现有提示词 (Prompt)
- **评估**：测试框架和成功标准
- **反模式**：识别并纠正常见错误
- **多模态**：视觉、嵌入和基于文件的提示词 (Prompt)

### 关键功能

1. **提示词 (Prompt) 分析**
   - 查看现有提示词 (Prompt)
   - 识别改进机会
   - 发现反模式和问题
   - 提出具体的改进建议

2. **提示词 (Prompt) 生成**
   - 从头开始创建新提示词 (Prompt)
   - 针对特定用例的设计
   - 确保清晰度和有效性
   - 优化一致性

3. **自定义指令**
   - 设计系统提示词 (Prompt)
   - 创建代理指令
   - 定义行为准则
   - 设置适当的约束

4. **最佳实践指南**
   - 解释提示词 (Prompt) 工程原理
   - 传授先进技术
   - 分享现实世界的例子
   - 提供实施指导

5. **测试与验证**
   - 开发测试用例
   - 定义成功标准
   - 评估提示词 (Prompt) 表现
   - 识别边缘情况

## 如何使用此技能

### 用于提示词 (Prompt) 分析
```
"Review this prompt and suggest improvements:
[YOUR PROMPT]

Focus on: clarity, specificity, format, and consistency."
```

### 用于提示词 (Prompt) 生成
```
"Create a prompt that:
- [Requirement 1]
- [Requirement 2]
- [Requirement 3]

The prompt should handle [use cases]."
```

### 自定义指令
```
"Design custom instructions for an agent that:
- [Role/expertise]
- [Key responsibilities]
- [Behavioral guidelines]"
```

### 用于故障排除
```
"This prompt isn't working well:
[PROMPT]

Issues: [DESCRIBE ISSUES]

How can I fix it?"
```

## 最佳实践总结

### 要做的 ✅
- 清晰具体
- 提供例子
- 指定格式
- 定义约束
- 彻底测试
- 记录假设
- 使用渐进式披露
- 处理边缘情况

### 不该做的事 ❌
- 含糊或模棱两可
- 假设理解
- 跳过格式规范
- 忽略边缘情况
- 过度指定约束
- 使用行话而不解释
- 硬编码值
- 忽略错误处理

## 限制

- **仅分析**：不执行代码或运行实际提示词 (Prompt)
- **无实时数据**：无法访问外部 API 或当前数据
- **基于最佳实践**：基于既定模式的建议
- **需要测试**：建议应通过实际用例进行验证
- **人类判断**：不会取代关键应用中的人类专业知识
