# 记忆系统 (Memory) 🧠

**超强记忆力，永不遗忘。**

您的代理拥有基础的内置内存。这项技能为其添加了无限且组织完美的记忆——并行互补，互不冲突。

## 运作原理

```
内置代理内存 (Built-in)          这项技能 (Skill) (~/memory/)
┌─────────────────────┐        ┌─────────────────────────────┐
│ MEMORY.md           │        │ 无限的分类存储               │
│ memory/ (每日日志)   │   +    │ 随心所欲的自定义结构         │
│ 基础召回能力         │        │ 完美的组织架构               │
└─────────────────────┘        └─────────────────────────────┘
         ↓                                  ↓
    代理基础功能                       其它所有内容
    (自动运行)                         (无限扩展)
```

**并非替代品。** 代理的内置内存将继续工作。此技能增加了一个并行系统，用于无限的有组织存储。

## 设置

首次使用时，请阅读 `setup.md` 以与用户共同配置记忆系统。关键决策点：
1. 他们需要哪些类别？
2. 是否需要从内置内存同步内容？
3. 他们希望如何查找信息？

## 何时使用

当用户需要超出基础代理内存的有组织长期存储时：详细的项目历史、庞大的联系人网络、决策日志、领域知识、各类收藏或任何随时间增长的结构化数据。

## 架构

记忆存储在 `~/memory/` 中——这是一个与内置代理内存分开的专用文件夹。

```
~/memory/
├── config.md              # 系统配置
├── INDEX.md               # 存储内容索引，查找入口
│
├── [用户定义]/             # 用户需要的类别
│   ├── INDEX.md           # 类别概览
│   └── {项目}.md           # 具体条目
│
└── sync/                  # 可选：从内置内存同步的内容
    └── ...
```

**用户定义类别。** 常见示例：
- `projects/` — 详细的项目上下文
- `people/` — 带有完整背景的联系人网络
- `decisions/` — 决策背后的权衡逻辑
- `knowledge/` — 领域专业知识、参考资料
- `collections/` — 书籍、食谱、各类收藏品

有关所有模板，请参阅 `memory-template.md`。

## 核心规则

### 1. 与内置内存分离

该系统位于 `~/memory/` 目录中。切勿修改：
- 代理的 `MEMORY.md`（工作区根目录）
- 代理的 `memory/` 文件夹（如果工作区中存在）

**并行，而非替代。** 两个系统协同工作。

### 2. 用户定义的结构

在设置过程中，询问用户想要存储什么。根据其需求创建类别：

| 用户说... | 创建 |
|-------------|--------|
| “我有很多项目” | `~/memory/projects/` |
| “我认识很多人” | `~/memory/people/` |
| “我想跟踪决策” | `~/memory/decisions/` |
| “我正在学习 [主题]” | `~/memory/knowledge/[主题]/` |
| “我收集 [东西]” | `~/memory/collections/[事物]/` |

**没有预设结构。** 只构建他们需要的东西。

### 3. 每个类别都有索引

每个文件夹都有一个 `INDEX.md` 列出其内容：

```markdown
# 项目索引

| 名称 | 状态 | 更新日期 | 文件 |
|------|--------|---------|------|
| Alpha | 活跃 | 2026-02 | alpha.md |
| Beta | 暂停 | 2026-01 | beta.md |

总计: 2 个活跃, 5 个已存档
```

保持索引文件短小（<100 个条目）。如果条目过多，请拆分为子类别。

### 4. 立即写入

当用户分享重要信息时：
1. 写入 `~/memory/` 中对应的文件
2. 更新该类别的 `INDEX.md`
3. 然后再回复用户

不要等待，不要批量处理，请立即写入。

### 5. 先搜索，后导航

查找信息的方法：
1. **先问：** “这是在 `~/memory/` 中还是在内置内存中？”
2. **搜索：** 在 `~/memory/` 中使用 grep 或语义搜索
3. **导航：** `INDEX.md` → 类别目录 → 特定文件

```bash
# 快速搜索
grep -r "关键字" ~/memory/

# 导航查找
cat ~/memory/INDEX.md           # 有哪些类别？
cat ~/memory/projects/INDEX.md  # 有哪些项目？
cat ~/memory/projects/alpha.md  # 具体项目详情
```

### 6. 从内置内存同步（可选）

如果用户想从内置内存复制某些信息：

```
~/memory/sync/
├── preferences.md    # 从内置同步
└── decisions.md      # 从内置同步
```

**同步是单向的：** 内置 → 此系统。切勿修改内置内存。

### 7. 通过拆分实现扩展

当某个类别变得过于庞大时：
- `INDEX.md` > 100 个条目 → 拆分为子类别
- 为每个子类别创建 `sub-INDEX.md`
- 根 `INDEX.md` 指向这些子类别

```
~/memory/projects/
├── INDEX.md           # “参见 active/, archived/”
├── active/
│   ├── INDEX.md       # 30 个活跃项目
│   └── ...
└── archived/
    ├── INDEX.md       # 200 个已存档项目
    └── ...
```

## 存储位置决策（与内置内存对比）

| 存储在这里 (`~/memory/`) | 保持内置 (Built-in) |
|------------------------------------|------------------|
| 详细的项目历史记录 | 当前项目状态 |
| 完整的联系人详情 | 主要联系人快速参考 |
| 所有决策的权衡推理 | 最近的决策 |
| 领域知识库 | 快速参考摘要 |
| 收藏品、库存列表 | — |
| 任何规模庞大的内容 | 总结摘要 |

**规则：** 内置内存用于快速获取上下文。此系统用于深度存储和大规模扩展。

## 查找信息

### 小型记忆库 (<50 个文件)
```bash
# Grep 速度足够快
grep -r "关键字" ~/memory/
```

### 大型记忆库 (50+ 个文件)
通过索引文件导航：
```
1. ~/memory/INDEX.md → 找到类别
2. ~/memory/{类别}/INDEX.md → 找到项目
3. ~/memory/{类别}/{项目}.md → 读取详情
```

### 巨型记忆库 (500+ 个文件)
使用语义搜索（如果可用）或多层索引：
```
~/memory/projects/INDEX.md → “web 项目在 web/ 目录”
~/memory/projects/web/INDEX.md → “找到 alpha 项目”
~/memory/projects/web/alpha.md → 读取详情
```

## 维护

### 每周维护 (5 分钟)
- 如果添加了新条目，更新 `INDEX.md` 文件
- 存档已完成或不再活跃的项目

### 每月维护 (15 分钟)
- 查看各类别的大小
- 拆分过大的类别
- 删除过时的条目

### 当记忆查找变慢时
- 检查 `INDEX.md` 的大小（保持在 100 行以内）
- 将大类别拆分为子类别
- 存档旧内容

## 常见陷阱

- **修改内置内存** → 绝对不要触碰代理的 `MEMORY.md` 或工作区的 `memory/`。本系统是并行的。

- **没有索引** → 如果没有 `INDEX.md`，查找内容就需要搜索所有文件。请始终维护索引。

- **单个类别过于庞大** → 一个文件夹里放 500 个项目会导致速度变慢。请拆分为子类别。

- **同步所有内容** → 不要复制所有的内置内存。只在这里同步需要组织整理的内容。

- **等待写入** → 用户分享信息时请立即写入，不要积压，不要批量处理。

## 安全与隐私

**数据位置：**
- 所有数据都在用户机器上的 `~/memory/` 中
- 无需外部服务
- 无网络请求

**此技能不会：**
- 擅自访问内置代理内存（仅在同步时读取）
- 向任何地方发送数据
- 存储凭据（切勿在记忆系统中存储密钥、密码等）
