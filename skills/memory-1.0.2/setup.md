# 设置 — 记忆系统

首次使用时请阅读本文。指导用户设置他们的个人记忆系统。

## 你的态度

你正在赋予他们超能力。无限的记忆，完美的组织，满足他们想要的任何东西。这是令人兴奋的——帮助他们看到可能性。

**重要提示：** 这与内置代理内存 (Memory) 分开。这是一个并行系统，对现有系统进行了补充。

## 对话流程

### 1. 解释这是什么

“我可以为你建立一个无限的记忆系统——与我的基础内存 (Memory) 分开。它适用于你想要长期存储的任何东西：项目、人员、决策、知识、收藏……任何对你来说重要的东西。

它不会干扰我通常记忆事物的方式。这是额外的、有组织的存储，可以根据您的需要进行扩展。”

### 2. 询问他们需要什么

“什么样的事情如果完美组织起来最有用？

人们使用的一些例子：
- **项目** — 每个项目的完整历史、决策、背景
- **人员** — 与您一起工作的每个人的详细资料
- **决策** — 为什么您选择方案 X 而不是 Y，以便您稍后记住
- **知识** — 您正在学习的内容、参考资料
- **收藏** — 书籍、食谱、想法、您收集的任何东西”

让他们告诉你，不要主观假设。

### 3. 询问同步事宜

“我的内置内存 (Memory) 已经跟踪了一些事情。您希望我将其中任何内容同步到这个新系统中吗？

例如，我可以复制：
- 您告诉我的偏好设置
- 我们做出的重要决策
- 主要联系人

或者我们可以重新开始，只添加新的东西。”

### 4. 创建结构

根据他们的回答，使用以下内容创建 `~/memory/` 目录：
- `config.md` — 他们的偏好设置
- `INDEX.md` — 根索引
- 为他们提到的每个类别创建文件夹
- 每个文件夹中创建 `INDEX.md`

### 5. 第一个条目

问：“你现在想让我记住什么？”

立即保存，向他们展示系统的有效性。

## 你需要保存的内容

**在 `~/memory/config.md` 中：**
```markdown
# 记忆配置

创建日期: YYYY-MM-DD
是否从内置内存同步: [是/否]

## 类别
- projects/ (项目)
- people/ (人物)
- [用户提到的其它类别]

## 偏好设置
- [他们希望如何查找事物]
- [组织维护的频率]
```

**在 `~/memory/INDEX.md` 中：**
```markdown
# 记忆索引

| 类别 | 描述 | 索引文件 |
|----------|-------------|-------|
| 项目 | 项目历史记录 | projects/INDEX.md |
| 人物 | 联系人网络 | people/INDEX.md |

最后更新日期: YYYY-MM-DD
```

## 当“完成”时

一旦你：
1. 创建了 `~/memory/` 及其类别目录
2. 设置了 `INDEX.md` 文件
3. 存储了一个真实的条目

...系统即告上线。它将通过正常使用不断成长。
