# 记忆模板

## 系统配置

创建 `~/memory/config.md`：

```markdown
# 记忆配置

创建日期: YYYY-MM-DD
所有者: [姓名]

## 同步设置
sync_from_builtin (从内置同步): false
sync_categories (同步类别): []

## 类别
- projects/ (项目)
- people/ (人物)
- decisions/ (决策)
- [custom]/ (自定义)

## 偏好设置
find_method (查找方法): navigate | search | both (导航 | 搜索 | 两者)
maintenance (维护周期): weekly | monthly (每周 | 每月)
```

## 同步文件夹（可选）

如果用户想从内置内存同步：

**`~/memory/sync/INDEX.md`**
```markdown
# 从内置内存同步

| 内容 | 来源 | 上次同步 | 文件 |
|------|--------|-----------|------|
| 偏好设置 | MEMORY.md | 2026-02-22 | preferences.md |
| 关键决策 | MEMORY.md | 2026-02-22 | decisions.md |

注意：这是单向同步。内置内存不会被修改。
```

## 索引大小限制

| 索引类型 | 最大条目数 | 超过时的处理方式 |
|------------|-------------|----------------|
| 根索引 (INDEX.md) | 20 个类别 | 极少发生，无需处理 |
| 类别索引 (INDEX.md) | 100 个项目 | 拆分为子类别 |
| 子类别索引 (INDEX.md) | 100 个项目 | 再次拆分 |

**拆分示例：**
```
projects/
├── INDEX.md              # "参见 active/, archived/"
├── active/
│   ├── INDEX.md          # 活跃项目
│   └── ...
└── archived/
    ├── INDEX.md          # 已存档项目
    └── ...
```
