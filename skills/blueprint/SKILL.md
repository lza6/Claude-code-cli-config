# 蓝图 — 构建计划生成器 (Blueprint — Construction Plan Generator)

将单行目标转换为分步构建计划，使任何编程代理都能直接上手执行。

## 何时使用

- 将大型功能拆分为多个具有清晰依赖顺序的 PR
- 规划跨越多个会话的重构或迁移
- 协调子代理之间的并行工作流
- 任何会话间上下文丢失可能导致返工的任务

**不要用于**：可在单个 PR 中完成的任务、少于 3 次工具调用的任务，或当用户说“直接做”时。

## 工作原理

Blueprint 运行 5 阶段管道：

1. **研究 (Research)** — 预检（git, gh auth, remote, 默认分支），然后读取项目结构、现有计划和记忆文件以收集上下文。
2. **设计 (Design)** — 将目标拆分为 PR 规模的步骤（通常为 3-12 个）。为每个步骤分配依赖边、并行/串行顺序、模型层级（最强 vs 默认）以及回滚策略。
3. **起草 (Draft)** — 在 `plans/` 中编写一个自包含的 Markdown 计划文件。每个步骤都包含上下文简报、任务列表、验证命令和退出标准 —— 因此新代理无需阅读前面的步骤即可执行任何一步。
4. **审查 (Review)** — 委托一个最强模型子代理（如 Opus）对照检查清单和反面模式目录进行对抗性审查。在最终确定前修复所有关键发现。
5. **注册 (Register)** — 保存计划，更新记忆索引，并向用户展示步骤计数和并行性摘要。

Blueprint 会自动检测 git/gh 的可用性。如果具备 git + GitHub CLI，它会生成完整的分支/PR/CI 工作流计划。如果不具备，它将切换到直接模式（就地编辑，不使用分支）。

## 示例

### 基础用法

```
/blueprint myapp "将数据库迁移到 PostgreSQL"
```

生成 `plans/myapp-migrate-database-to-postgresql.md`，步骤如下：
- 第 1 步：添加 PostgreSQL 驱动和连接配置
- 第 2 步：为每个表创建迁移脚本
- 第 3 步：更新仓储层以使用新驱动
- 第 4 步：添加针对 PostgreSQL 的集成测试
- 第 5 步：移除旧的数据库代码和配置

### 多代理项目

```
/blueprint chatbot "将 LLM 提供商提取到插件系统中"
```

生成一个包含并行步骤（如在插件接口步骤完成后，“实现 Anthropic 插件”和“实现 OpenAI 插件”并行运行）、模型层级分配（接口设计步骤使用最强模型，实现使用默认模型）以及每步后验证不变量（如“所有现有测试通过”、“核心代码中没有提供商导入”）的计划。

## 关键特性

- **冷启动执行** — 每个步骤都包含自包含的上下文简报。无需预先了解上下文。
- **对抗性审查门禁** — 每个计划都由最强模型子代理对照涵盖完整性、依赖正确性和反面模式检测的检查清单进行审查。
- **分支/PR/CI 工作流** — 内置于每个步骤中。当缺少 git/gh 时，平滑降级到直接模式。
- **并行步骤检测** — 依赖图识别没有共享文件或输出依赖关系的步骤。
- **计划变更协议** — 步骤可以通过正式协议和审计轨迹进行拆分、插入、跳过、重排序或放弃。
- **零运行时风险** — 纯 Markdown 技能。整个仓库仅包含 `.md` 文件 —— 无钩子、无 shell 脚本、无执行代码、无 `package.json`、无构建步骤。安装或调用时，除了 Claude Code 原生的 Markdown 技能加载器外，不运行任何内容。

## 安装

此技能随 Everything Claude Code (ECC) 一起提供。安装 ECC 后无需单独安装。

### 完整 ECC 安装

如果您是从 ECC 仓库签出的，请验证技能是否存在：

```bash
test -f skills/blueprint/SKILL.md
```

以后更新时，请在更新前审查 ECC 的差异：

```bash
cd /path/to/everything-claude-code
git fetch origin main
git log --oneline HEAD..origin/main       # 更新前审查新提交
git checkout <reviewed-full-sha>          # 固定到特定的已审查提交
```

### 独立引入安装

如果您仅在完整 ECC 安装之外引入此技能，请将 ECC 仓库中经过审查的文件复制到 `~/.claude/skills/blueprint/SKILL.md`。引入的副本没有 git 远程仓库，因此请通过重新复制经过审查的 ECC 提交中的文件来更新，而不是运行 `git pull`。

## 要求

- Claude Code（用于 `/blueprint` 斜杠命令）
- Git + GitHub CLI（可选 —— 启用完整的分支/PR/CI 工作流；Blueprint 会检测缺失并自动切换到直接模式）
