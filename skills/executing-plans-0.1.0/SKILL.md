# 执行计划

## 概述

加载计划，严格审查，分批执行任务，报告批间审查。

**核心原则：** 带有检查点的批量执行，以供架构师审查。

**开始时宣布：**“我正在使用执行计划技能来实施这个计划。”

## 过程

### 第 1 步：加载并审查计划
1. 阅读计划文件
2. 批判性地审查——找出有关计划的任何问题或疑虑
3. 如果有疑虑：在开始之前向您的人类伴侣提出
4. 如果没有问题：创建 TodoWrite 并继续

### 第 2 步：执行批处理
**默认：前 3 个任务**

对于每个任务：
1. 标记为 in_progress
2. 严格执行每一步（计划中有细小的步骤）
3. 按照规定运行验证
4. 标记为已完成

### 第 3 步：报告
批次完成后：
- 显示已实施的内容
- 显示验证输出
- 说：“准备好反馈。”

### 第 4 步：继续
根据反馈：
- 如果需要，应用更改
- 执行下一批
- 重复直到完成

### 第 5 步：完成开发

所有任务完成并验证后：
- 宣布：“我正在使用完成开发分支技能来完成这项工作。”
- **所需的子技能：** 使用超能力：完成开发分支
- 遵循该技能来验证测试、提出选项、执行选择

## 何时停下来寻求帮助

**在以下情况下立即停止执行：**
- 在批次中遇到阻碍（缺少依赖性、测试失败、指令不清楚）
- 计划存在严重缺陷，无法启动
- 你不明白指令
- 验证多次失败

**要求澄清而不是猜测。**

## 何时重新访问之前的步骤

**返回审查（第 1 步）当：**
- 搭档根据你的反馈更新了计划
- 基本方法需要重新思考

**不要强行通过阻碍** - 停下来询问。

## 记住
- 首先严格审查计划
- 严格遵循计划步骤
- 不要跳过验证
- 计划中提到的参考技能
- 批次之间：只需报告并等待
- 受阻时停止，不要猜测
