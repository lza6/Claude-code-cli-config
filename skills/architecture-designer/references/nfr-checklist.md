# 非功能性需求检查清单

## NFR 类别

### 可扩展性 (Scalability)

| 问题 | 常见目标 |
|----------|----------------|
| 预期的并发用户数？ | 100 / 1K / 10K / 100K |
| 每秒请求数 (RPS)？ | 10 / 100 / 1000 / 10000 |
| 数据量？ | GB / TB / PB |
| 增长率？ | 每年 10% / 50% / 100% |
| 峰值 vs 平均负载？ | 2x / 5x / 10x |

### 性能 (Performance)

| 问题 | 常见目标 |
|----------|----------------|
| API 响应时间？ | < 100ms / 200ms / 500ms (p95) |
| 页面加载时间？ | < 1s / 2s / 3s |
| 数据库查询时间？ | < 10ms / 50ms / 100ms |
| 批处理吞吐量？ | 1K / 10K / 100K 记录/小时 |

### 可用性 (Availability)

| 目标 | 每年停机时间 | 使用场景 |
|--------|---------------|----------|
| 99% | 3.65 天 | 内部工具 |
| 99.9% | 8.76 小时 | 商业应用 |
| 99.95% | 4.38 小时 | 电子商务 |
| 99.99% | 52.6 分钟 | 金融系统 |
| 99.999% | 5.26 分钟 | 生命关键型系统 |

### 安全性 (Security)

| 问题 | 考虑因素 |
|----------|----------------|
| 是否需要身份验证？ | JWT, OAuth, SAML, MFA |
| 授权模型？ | RBAC, ABAC, ACL |
| 数据敏感性？ | 公开、内部、机密、PII (个人身份信息) |
| 合规性要求？ | GDPR, HIPAA, PCI DSS, SOC 2 |
| 加密需求？ | 静态加密、传输中加密、端到端加密 |

### 可靠性 (Reliability)

| 问题 | 考虑因素 |
|----------|----------------|
| 可接受的数据丢失？ | RPO: 0 / 1 小时 / 24 小时 |
| 恢复时间目标？ | RTO: 1 小时 / 4 小时 / 24 小时 |
| 备份频率？ | 实时 / 每小时 / 每日 |
| 灾难恢复？ | 单区域 / 多区域 |

### 可维护性 (Maintainability)

| 问题 | 考虑因素 |
|----------|----------------|
| 部署频率？ | 每日 / 每周 / 每月 |
| 部署策略？ | 蓝绿部署、金丝雀发布、滚动更新 |
| 监控需求？ | 日志、指标、追踪、告警 |
| 值班要求？ | 24/7、工作时间 |

### 成本 (Cost)

| 问题 | 考虑因素 |
|----------|----------------|
| 基础设施预算？ | $/月、$/用户、$/请求 |
| 运维预算？ | 维护所需的全职人力 (FTE) |
| 成本优化？ | 预留实例、抢占式实例 (Spot) |
| 成本告警？ | 通知阈值 |

## 模板

```markdown
## 非功能性需求

### 性能
- API 响应时间：< 200ms p95
- 页面加载时间：< 2s
- 数据库查询时间：< 50ms

### 可扩展性
- 并发用户数：10,000
- 每秒请求数 (RPS)：1,000
- 数据量：1TB

### 可用性
- 目标：99.9% (每年停机时间 8.76 小时)
- RPO：1 小时
- RTO：4 小时

### 安全性
- 身份验证：带有刷新令牌的 JWT
- 授权：基于角色 (管理员、用户、访客)
- 合规性：GDPR, SOC 2

### 可观测性
- 日志：结构化 JSON 发送到 ELK
- 指标：Prometheus + Grafana
- 追踪：OpenTelemetry
- 告警：PagerDuty 集成
```
