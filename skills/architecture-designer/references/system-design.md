# 系统设计模板

## 设计模板

```markdown
# 系统：{系统名称}

## 需求

### 功能需求
- [系统必须做的事情]
- [核心功能和能力]

### 非功能性需求
- **性能**: 响应时间 < 200ms p95
- **可用性**: 99.9% 正常运行时间 (每年停机 8.76 小时)
- **可扩展性**: 支持 10,000 个并发用户
- **安全性**: 需要符合 PCI DSS 合规性

### 约束条件
- 预算：基础设施每月 $X
- 时间线：3 个月内交付 MVP
- 团队：5 名后端，3 名前端工程师

## 高层架构设计

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   客户端    │────▶│  API 网关   │────▶│    服务     │
│   (Web)     │     │   (Kong)    │     │  (Node.js)  │
└─────────────┘     └─────────────┘     └─────────────┘
                           │                   │
                           ▼                   ▼
                    ┌─────────────┐     ┌─────────────┐
                    │  身份验证   │     │   数据库    │
                    │   (Auth0)   │     │ (PostgreSQL)│
                    └─────────────┘     └─────────────┘
```

## 组件详情

### API 层
- 技术栈：Node.js 配合 Express/NestJS
- 职责：请求路由、验证、身份验证
- 扩展：通过负载均衡器水平扩展

### 数据层
- 主数据库：PostgreSQL (处理事务、关系)
- 缓存：Redis (存储会话、热点数据)
- 存储：S3 (文件、图像)

### 外部服务
- 身份验证：Auth0 (SSO, MFA)
- 邮件服务：SendGrid (交易类邮件)
- 监控：Datadog (APM, 日志)

## 关键决策

| 决策 | 理由 |
|----------|-----------|
| 选用 PostgreSQL 而非 MongoDB | 需要处理关系数据和 ACID 事务 |
| 选用 Redis 进行缓存 | 需要亚毫秒级延迟 |
| 选用 Auth0 而非自定义方案 | 降低安全风险 |

## 扩展策略

### 当前阶段 (MVP)
- 单区域部署
- ALB 后挂载 2 个 API 实例
- 单个 RDS 实例

### 未来阶段 (10 倍增长)
- 配合 CDN 进行多区域部署
- API 自动扩展 (2-10 个实例)
- RDS 读取副本

## 安全考虑
- 所有流量通过 TLS 1.3 传输
- JWT 令牌有效期为 15 分钟
- 速率限制：每位用户 100 请求/分钟
- 使用 WAF 防御常见攻击

## 故障模式

| 故障 | 影响 | 缓解策略 |
|---------|--------|------------|
| 数据库宕机 | 全面停服 | 多可用区 (Multi-AZ) 故障转移 |
| 缓存宕机 | 性能下降 | 降级到数据库查询 |
| 认证服务宕机 | 无法新登录 | 缓存有效令牌 |
```
