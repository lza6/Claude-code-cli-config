# 配置审计基线

OpenClaw 环境评估的配置安全基线。

## 审计命令

```bash
openclaw security audit --deep
```

## 检查类别

### 1. 网关认证
**检查内容：**
- 网关是否启用？
- 是否需要身份验证？
- 是否允许匿名连接？

**风险指标：**
- 网关无需授权即可打开 → 🔴 高危
- 弱密码/默认凭据 → 🔴 高危
- 正确的令牌/Cookie 身份验证 → ✅ 通过

### 2. 网络暴露
**检查内容：**
- 绑定地址（127.0.0.1 vs 0.0.0.0）
- 端口暴露
- UPnP/NAT-PMP 设置

**风险指标：**
- 暴露于互联网并绑定到 0.0.0.0 → 🔴 高危
- 仅本地主机绑定 → ✅ 通过
- mDNS/Bonjour 广播 → ⚠️ 低危

### 3. 工具爆炸半径
**检查内容：**
- 默认审批设置
- 工具权限范围
- 危险工具的自动审批

**风险指标：**
- Bash/exec 自动审批 → 🔴 高危
- 工作区外文件写入自动审批 → 🔴 高危
- 所有工具均需审批 → ✅ 通过

### 4. 浏览器控制
**检查内容：**
- 是否启用了远程浏览器操作？
- 无头浏览器权限
- 屏幕截图/捕获功能

**风险指标：**
- 不受限制的浏览器控制 → 🔴 高危
- 需要用户确认 → ⚠️ 中危
- 禁用/未配置 → ✅ 通过

### 5. 文件系统权限
**检查内容：**
- 工作区边界强制执行
- 敏感目录访问（ssh、aws 等）
- 文件权限要求

**风险指标：**
- 无需审批即可读取 ~/.ssh → 🔴 高危
- 可以写入系统目录 → 🔴 高危
- 仅限工作空间 → ✅ 通过

### 6. 开放房间/协作
**检查内容：**
- 开放房间设置
- 未经授权的加入能力
- 访客权限

**风险指标：**
- 任何人无需邀请即可加入 → ⚠️ 中危
- 适当的访问控制 → ✅ 通过
